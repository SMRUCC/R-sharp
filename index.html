GCModeller API Documents

GCModeller is original written in VisualBasic.NET language, API export to the Shoal Language through the reflection attributes:

  • Microsoft.VisualBasic.CommandLine.Reflection.ExportAPIAttribute
  • Microsoft.VisualBasic.Scripting.MetaData.PackageNamespace
    • And then you can using these attributes to build your own API for shoal language on the Microsoft .NET Platform by using language VisualBasic/C#/F#.

    Microsoft.VisualBasic.Scripting.MetaData.Cite attribute can be used on your API namespace to specific which the scientific article should be cites if you would like to publish your tools package in the GCModeller library to helps other researchers.

    The GCModeller API for shoal language was documented on the GCModeller help library at here: gcmodeller.org/library

    Shoal Programming Features

  • Memory pipeline in GCModeller

    • The memory pipeline was used in the GCModeller, which can save the times on the large data IO between the CLI tools when you are programming hybrids the .NET export API with the cmd shell in the shoal language. Usually when the data size is large than 1GB, that using the memory pipeline can save much of time on the file system IO between the analysis tools in the GCModeller. The upbound of the data size that in the shoal shell memory pipeline supported is up to 1TB.

  • The original R language hybrids

    • Shoal shell language can hybrids programing with R language from the plugin of RDotNET.
